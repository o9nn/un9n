## Usage Guidelines

### For Avatar Development

1. **Current Focus**: Continue developing avatar systems (Live2D SDK, 3D assets, personality)
2. **Pattern Awareness**: Use PatternAwareAvatarComponent to validate design decisions
3. **Quality Metrics**: Monitor "Quality Without a Name" score (target: >0.7)
4. **Documentation**: Refer to pattern language docs when designing new features

### For Future AGI-OS Integration

1. **Review Roadmap**: Start with INTEGRATION_ROADMAP.md for complete plan
2. **Phase 1 Priority**: Begin with 9P interface layer (3 weeks)
3. **Incremental Approach**: Complete each phase before proceeding
4. **Testing**: Validate each integration point thoroughly

### For Pattern Language Application

1. **Design Principle**: Every feature should strengthen existing centers
2. **Quality Check**: Calculate pattern properties before and after changes
3. **Wholeness**: Ensure changes increase overall system coherence
4. **Living Structure**: Aim for aliveness, not just functionality

## Development Principles

### From Pattern Language

- **Start with the Whole**: Consider impact on entire avatar system
- **Preserve Centers**: Strengthen existing strong centers
- **Iterate Incrementally**: Small changes that improve the whole
- **Test for Aliveness**: Does it make the avatar feel more alive?
- **Maintain Coherence**: Every addition strengthens existing patterns

### From AGI-OS Architecture

- **9P-First**: Design with 9P filesystem interface in mind
- **Message Coalescing**: Batch operations for performance
- **AtomSpace Integration**: Plan for persistent knowledge storage
- **Phased Implementation**: Build in layers, validate each layer
